## 1. Overview

The control electronics subsystem manages all charger intelligence: session orchestration, protocol communication, safety supervision, sensor acquisition, and network connectivity. These components operate on low-voltage rails (5V, 12V, 24V) generated by the auxiliary power supply.

The main controller is a [[research/04 - Phytec SBC Replacement|Phytec phyCORE-AM62x]] SOM with 3x native CAN FD interfaces, eliminating the need for external CAN bridge modules. See [[research/04 - Phytec SBC Replacement|04 - Phytec SBC Replacement]] for the selection rationale.

## 4. Notes

- The Phytec SBC runs the EVerest framework including OCPP 2.0.1 client, ISO 15118 stack, session management, and energy management on a Yocto Linux BSP with official EVerest integration
- 3x native CAN FD interfaces connect directly to power modules (CAN #1), safety supervisor (CAN #2), and HVAC (CAN #3) — no external CAN bridge modules required
- The safety controller operates independently from the main controller and handles all time-critical safety functions (contactor control, OCP/OVP, pre-charge sequencing)
- Two PLC controllers are required for a dual-connector configuration (CCS2 + NACS or 2× CCS2)
- The IO controller handles auxiliary sensor inputs and relay outputs on the DIN rail panel
- Industrial temperature range (-40°C to +85°C), secure boot (TrustZone), and 15+ year availability guarantee
